C的预处理

预处理的概念

预处理主要在编译之前处理程序中的特殊行

预处理指令包括： 宏定义、文件包含和条件编译

c程序 -> 预处理器 -> 修改后的c程序 -> 编译器 -> 目标代码

预处理器是一个小软件，在编译前处理c程序

预处理操作

<1> 宏定义。 #define指令定义一个宏， #undef指令删除一个宏

<2> 文件包含。 #include指令将一个指定文件内容包含到程序中

<3> 条件编译。 #if、#ifdef、#ifndef、#elif、#else和#endif指令根据测试条件来确定是否将一段程序文本包含到程序中

规则

<1> 以#开始

<2> 换行符作为结尾，’'可延续到下一行

<3> 可间隔任意空格、制表符，可放在程序任意位置，一般放在文件开始位置，可与注释同行

#define

#define 标识符 替换文本 //符号常量

#define 宏名(x1,x2,..) 替换列表 //带参数的宏定义

参数带上括号程序更安全

带参数的宏定义不是函数

宏定义没有类型检查

#include

格式1：#include “文件名” 优先查找工作目录，其次查找预先指定的编译器和系统目录。

格式2：#include <文件名> 在预先指定的编译器和系统目录查找

编译多文件时，无需编译.h文件，因为编译.c文件时会自动编译头文件的内容

编译生成一个包含目标代码的文件，称为目标文件，windows系统中扩展名为.obj

链接器把目标文件和库函数代码结合在一起生成可执行程序

链接器可以解决外部引用问题(extern 外部函数)

条件编译

条件编译可以用来保护头文件(文件嵌套无意间包含一个头文件多次)

源文件包含函数定义和函数调用

头文件包含类型定义、函数原型声明等

C++学习笔记

发表于 2021-12-14

4.5 共用体 （union）

用法和类相同

长度为最大成员的长度 共用同一块内存

匿名共用体

类似于同一起始地址有多个成员，但同时只能有一个成员活跃

常用于操作系统数据结构和硬件数据结构 节约内存

4.6 枚举 （enum）

创建符号常量的方式，可代替const

为了获取最大的移植性，将非enum值赋给enum变量视为error

enum为整型，可自动转为int

int要强制类型转换才能变成enum变量的值

可选择性赋值，默认第一个为0，以后每个枚举量增加1

可以创建多个相同值的枚举量

取址范围

上限为 最大值所在2的幂次的上限减1，120上限为127

下限为 最小值大于0，则取零；否则与上限相同，取负值，-6下限为-7

重载、覆盖和隐藏

发表于 2021-12-14 更新于 2021-12-18

重载

同一个类中的不同函数

<1> 相同的范围（在同一个类中）

<2> 函数名相同

<3> 参数不同

覆盖

派生类函数覆盖基类函数

<1> 不同范围（基类和派生类）

<2> 函数名相同

<3> 参数相同

<4> 基类函数带virtual

隐藏

派生类函数屏蔽了同名的基类函数

<1> 同名，参数不同，有无virtual基类函数都被隐藏

<2> 同名，同参，无virtual时基类函数被隐藏
